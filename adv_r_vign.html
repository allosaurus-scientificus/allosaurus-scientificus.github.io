Atomic

Using c() to make 4 types of atomic vectors.

v_num <- c(1, 2.5, 4.5)  # numeric == double
v_int <- c(1L, 3L, 5L)  #  note L suffix for int
v_log <- c(TRUE, FALSE, T, F)
v_chr <- c("hello", "world")

#  typeof(v_num)
#  typeof(v_int)
#  typeof(v_log)
#  typeof(v_chr)

Vectors are flat c(1, c(2, c(3, 4))) same as c(1, 2, 3, 4).

In addtion to typeof can use

  • is.double()
  • is.integer()
  • is.logical()
  • is.character()
  • is.atomic()

Turns out is.numeric not the same as is.double.

Attributes

Additional, arbitrary metadata.

attr(x, "my_arbitrary_attr") <- "example of an arbitrary attribute for vector x"

##  - attr(*, "my_arbitrary_attr")= chr "example of an arbitrary attribute for vector x"

Attributes (metadata) often lost when modifying a vector, except:

  • Names
  • Dimensions
  • Class

Names

Vector names don’t have an obvious use-case for me. Don’t have to be unique, but maybe should be for subsetting.

x <- c(a = 1, b = 2, c = 3)
x
## a b c 
## 1 2 3

Factors

Factors built on integer vectors with 2 attributes: class() == factor and levels(). That is important: a factor is really an integer with attributes to level. Beware when trying string methods on factors: sometimes they will coerce or sometimes they err.

You don’t need to have all the possible levels() in your data to set:

x <- factor(rep("m", 3), levels = c("m", "f"))
table(x)
## x
## m f 
## 3 0

Data Frames

Data frame == list of equal length vectors:

  • 2d
  • has
    • names() and colnames()
    • rownames()
    • legnth() == ncol() (the list length)
    • nrow() for rows

Don’t try to use cbind() to make a data frame, do it directly:

df = data.frame(x = 1:3, y = c("a", "b", "c"), stringsAsFActors = FALSE)

A data frame is a list, so just like before, you can next a list inside a list (data frame):

df <- data.frame(x = 1:3)

Data Types

Starting with [

Atomic Vectors

##  decimal marks original position
x <- c(2.1, 4.2, 3.3, 5.4)

Positive integers return whatever is at that position.

x[c(3, 1)]
## [1] 3.3 2.1
order(x)
## [1] 1 3 2 4
x[order(x)]
## [1] 2.1 3.3 4.2 5.4
x[c(1, 1, 1, 3)]
## [1] 2.1 2.1 2.1 3.3

Negative integers omit the position. You can’t mix pos and neg in single subset.

Logical vectors

x[x > 3]
## [1] 4.2 3.3 5.4
##  recycling
x[c(TRUE, FALSE)]
## [1] 2.1 3.3

Nothing, as in x[], returns original vector.

Zero, as in x[0] returns a vector of length 0.

If vector named then can use the names:

y <- setNames(x, letters[1:4])

y[c("d", "a")]
##   d   a 
## 5.4 2.1

Lists

[ returns a list, always; [[ and $ returns components of lists.

Data Frames

Data frame == matrix, list mashups

If single vector, behave like lists; if two vectors, behave like matrix

d <- data.frame(x = 1:3, y = 1:3, z = letters[1:3])

d[d$x == 2, ]
##   x y z
## 2 2 2 b
##  list like
d[c("x", "z")]
##   x z
## 1 1 a
## 2 2 b
## 3 3 c
##  matrix like
d[, c("x", "z")]
##   x z
## 1 1 a
## 2 2 b
## 3 3 c
##  matrix simplifies dimensions by default
##  matrix like
str(d[ , "y"])
##  int [1:3] 1 2 3
##  list way
str(d["y"])
## 'data.frame':    3 obs. of  1 variable:
##  $ y: int  1 2 3

fx_diag <- function(m) {
  stopifnot(is.matrix(x), nrow(m) == ncol(m))
  c <- ncol(m)
  return(m[matrix(seq_len(c), nrow = c, ncol = 2)])
}
